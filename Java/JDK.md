JDK содержит JRE и инструменты для разработчиков.
JRE содержит JVM и набор стандартных Java-библиотек.
JVM.

### Пакеты


Механизм для организации классов в связанные группы. Избежать конфликтов имен, разные пакеты классы с одинаковыми именами.

### Наименование

Имя пакета ```io.hexlet``` директории файл ```io/hexlet/Course.java```.
Префикс компании или разработчика - имя домена (```hexlet.io```) в обратном порядке (```io.hexlet```).

### Импорт

Класс из другого пакета импортировать ```import io.hexlet.model.User;``` - обращаться к классу по имени без полного пути.

Импорт всех классов из пакета ```import java.util.*;``` - захламляет пространство имен, и классы из разных пакетов пересечься по именам.

Импорт статических методы ```import static java.lang.Math.*;```. Не указывать имя класса при вызове статического метода ```double result = sqrt(16);```.

Если два класса из разных пакетов с одинаковым именем - полное имя для одного из них ```var name = io.hexlet.utils.User.getUserName();```.


Каждый класс в Java обязательно находится в пакете.
Избежать конфликтов имён, в проекте два класса Main или имя совпадает с именем класса библиотеки — в разных пакетах не будут мешать друг другу.
