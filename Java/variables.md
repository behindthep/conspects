### Конвенртация типов

Java — строго типизированный язык. у компьютера “коробочка” для числа, засунуть туда текст — вставить вилку в USB-разъём. Компилятор не простит.

Компилятор не умеет автоматически преобразовывать строку в число, Нужно привести строку к числу явно.

int number = 42;
String str = String.valueOf(number);  // str == "42"
String.valueOf() преобразует число в такую строку, которая соответствует типу переданного в нее объекта.

String str = "" + number;

все переменные, объекты, выражения в Java можно преобразовать к типу String. это автоматически происходит, когда мы складываем тип String с каким-нибудь другим типом.

String code = "Yo";
String message = "Hello! " + number + code; //  Hello! 10Yo

Операция сложения выполняется слева направо:
int a = 5;
String name = a + a + "1" + a;      // строку 1015
Порядок выполнения: ((a + a) + "1") + a


Преобразование строки (только из цифр) в число. Для отрицательных значений это тоже работает

int number2 = Integer.parseInt("321");
int number3 = Integer.parseInt("321" + 0); //число 3210
int number4 = "321"; //  Не скомпилируется: переменная int, а значение string


### Память

Каждая программа (в том числе на Java) перед выполнением загружается в оперативную память. В оперативной памяти находится код программы (который исполняется процессором) и данные программы (которые в память помещает сама программа).
Страница в Excel из ячеек, и у каждой ячейки есть уникальный номер (A1, A2, ... B1, B2). Зная номер ячейки, можно записать в неёо значение или получить значение, которое там хранится. Память компьютера устроена похоже.

Программа и её данные во время работы хранятся в памяти. память компьютера представлена в виде ячеек – байт. У ячейки есть её уникальный номер – 0, 1, 2, 3, ... (нумерация с нуля) - адрес. Зная номер ячейки, можем сохранить в эту ячейку данные. Или взять их из неё. В одних ячейках хранится код программы – набор команд для процессора, в других – данные этой программы.

Процессор умеет исполнять команды из загруженной в память программы. Почти все команды процессора — взять данные из некоторых ячеек → сделать с ними что-то → результат поместить в другие ячейки.
Объединяя сотни простых команд, получаем сложные и полезные.

Когда в коде программы объявляется переменная, ей выделяется кусочек ещё не использованной памяти (несколько байт). При объявлении переменной указать тип информации, которую программа хранить в ней: числа, текст... Без знания типа информации не ясно, какого размера блок памяти выделить под переменную.

На заре компьютернов, программы работали с номерами ячеек памяти, но потом ячейкам стали давать имена (имя переменной).


4 типа данных для хранения целых чисел byte, short, int и long.
Тип	Размер, байт	Происхождение имени
byte  1  занимает один байт памяти
short  2  Short Integer
int  4
long  8  Long Integer

2 вещественных типа — float и double:
Тип	Размер, байт	Происхождение имени
float  4  Floating Point Number
double  8  Double Float

Когда выполнение программы доходит до команды создания переменной, ей выделяется область памяти размером, в зависити от типа переменной.
Адрес переменной - адрес первой ячейки выделенного под неё блока памяти.
Java-программам запрещено напрямую обращаться к памяти. Работа с памятью только через Java-машину.


Тип String может хранить большие объёмы данных, не просто тип данных, а объект.
Данные типа String (текст) помещаются в спец объект, под который выделяется память, а уже адрес этого объекта помещается в переменную, под которую тоже выделяется память.

Переменная b int занимает 4 байта и хранит значение 10,555. Запятая — не дробная часть числа, а разделение разрядов.
Переменная str String хранит значение D12 — адрес первой ячейки объекта, содержащего текст.
Объект типа String (содержащий текст) хранится отдельным блоком памяти. Адрес его первой ячейки хранится в переменной str.


int a = 10;
int b = a;
b = 20;
System.out.println(a); // 10
в памяти создаются две ячейки: для a, для b. b = a; копируется значение (10), а не сам "ящик". Изменения в b не влияют на a.

String s1 = "Hello";
String s2 = s1;
s2 = s2 + " World";
System.out.println(s1); // "Hello"
System.out.println(s2); // "Hello World"
s1 и s2 ссылаются на одну и ту же строку "Hello" до момента изменения. Но когда выполняем s2 = s2 + " World"; для s2 создаётся новая строка "Hello World", а s1 по-прежнему смотрит на старую строку "Hello".

переменной выделили 100 байт памяти и есть X – адрес первого байта, какие адреса у всех байтов? X+0, X+1, X+2, ..., X+99. группу байтов с индексами 0, 1, 2, 99.
