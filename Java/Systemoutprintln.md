### Чтение с консоли

Как и для вывода (System.out), для ввода данных тоже есть объект — System.in. считывать данные с клавиатуры, по одному символу за раз.

классом, в паре с объектом System.in.

Класс java.util.Scanner считывать данные из консоль, файлы, интернет. считывал данные с клавиатуры - передать ему объект System.in в качестве параметра – источника данных. объект типа Scanner разберется, что с ним делать.

Scanner console = new Scanner(System.in);
String name = console.nextLine(); // Читаем с клавиатуры строку текста.
int age = console.nextInt(); // Читаем с клавиатуры число. Если данные, невозможно преобразовать в целое число - ошибка.
double number = console.nextDouble(); // проверяет, что введенные данные можно преобразовать в число double.

говорим Java-машине: создай новый объект (new) типа Scanner и передай в него в качестве параметра то, из чего новосозданный объект Scanner будет брать данные — объект System.in.

переменная console типа Scanner хранила у себя внутри ссылку на объект типа Scanner. вызвать методы объекта, на который ссылается переменная.

ввести несколько чисел в одной строке, разделив их пробелами


console.nextLine(); // Поглощаем оставшийся символ новой строки 
"съесть" (проглотить) символ новой строки (\n), оставшийся в буфере после вызова console.nextInt().

console.nextInt() читает целое число из входных данных, но не считывает символ новой строки, который вводится после числа нажатием Enter. символ остаётся во входном буфере.
След вызов console.nextLine() считывает весь оставшийся в буфере текст до символа новой строки, а в данном случае — только сам символ новой строки. 
Это необходимо, чтобы очистить буфер, иначе след вызов console.nextLine() сразу же прочитает пустую строку (оставшийся после числа символ новой строки), и пользовательский ввод не будет корректным.
