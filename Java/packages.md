## Packages

Механизм для организации классов в логически связанные группы. Избежать конфликтов имен, разные пакеты могут иметь классы с одинаковыми именами.

### Наименование

Имя пакета ```io.hexlet``` соответствует директории в которой файл ```io/hexlet/Course.java```.
Префикс компании или разработчика - имя домена (```hexlet.io```) в обратном порядке (```io.hexlet```).

### Импорт

Класс из другого пакета импортировать ```import io.hexlet.model.User;``` - обращаться к классу по имени без полного пути.

Импорт всех классов из пакета ```import java.util.*;``` - захламляет пространство имен, и классы из разных пакетов могут пересечься по именам.

Импорт статических методы ```import static java.lang.Math.*;```. Не указывать имя класса при вызове статического метода ```double result = sqrt(16);```.

Если два класса из разных пакетов с одинаковым именем - использовать полное имя для одного из них ```var name = io.hexlet.utils.User.getUserName();```.
